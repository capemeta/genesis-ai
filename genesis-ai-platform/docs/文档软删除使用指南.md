# 文档软删除功能使用指南

## 概述

文档软删除功能提供了安全的文档删除机制，支持：
- ✅ 软删除（可恢复）
- ✅ 回收站查看
- ✅ 文档恢复
- ✅ 物理删除（管理员）
- ✅ 定期自动清理

## 数据库设计

### 字段说明

```sql
-- documents 表新增字段
is_deleted BOOLEAN DEFAULT false NOT NULL  -- 软删除标志
deleted_at TIMESTAMPTZ                     -- 删除时间
deleted_by_id UUID                         -- 删除人ID
deleted_by_name VARCHAR(255)               -- 删除人名称
```

### 索引优化

```sql
-- 部分索引：只索引未删除的文档（提升查询性能）
CREATE INDEX idx_docs_active ON documents (tenant_id, owner_id) 
WHERE is_deleted = false;

-- 索引已删除的文档（用于回收站查询和定期清理）
CREATE INDEX idx_docs_deleted ON documents (tenant_id, deleted_at) 
WHERE is_deleted = true;
```

## API 接口

### 1. 软删除文档

**接口**: `POST /api/v1/documents/soft-delete`

**请求体**:
```json
{
  "document_id": "uuid-xxx"
}
```

**响应**:
```json
{
  "success": true,
  "message": "文档已移至回收站",
  "data": {
    "document_id": "uuid-xxx",
    "document_name": "产品手册.pdf"
  }
}
```

**特点**:
- 标记为已删除，但数据仍保留
- 可以通过回收站恢复
- 不删除物理文件
- 只有所有者可以删除

### 2. 恢复文档

**接口**: `POST /api/v1/documents/restore`

**请求体**:
```json
{
  "document_id": "uuid-xxx"
}
```

**响应**:
```json
{
  "success": true,
  "message": "文档已恢复",
  "data": {
    "document_id": "uuid-xxx",
    "document_name": "产品手册.pdf"
  }
}
```

**特点**:
- 从回收站恢复文档
- 只有所有者可以恢复

### 3. 查看回收站

**接口**: `GET /api/v1/documents/recycle-bin?page=1&page_size=20`

**响应**:
```json
{
  "success": true,
  "data": [
    {
      "id": "uuid-xxx",
      "name": "产品手册.pdf",
      "file_type": "PDF",
      "file_size": 1024000,
      "deleted_at": "2026-02-12T10:30:00Z",
      "deleted_by_name": "张三",
      "created_at": "2026-01-01T08:00:00Z"
    }
  ],
  "total": 5,
  "page": 1,
  "page_size": 20
}
```

**特点**:
- 只显示当前用户的已删除文档
- 支持分页
- 按删除时间倒序排列

### 4. 物理删除文档（谨慎使用）

**接口**: `POST /api/v1/documents/permanent-delete`

**请求体**:
```json
{
  "document_id": "uuid-xxx"
}
```

**响应**:
```json
{
  "success": true,
  "message": "文档已彻底删除",
  "data": {
    "document_id": "uuid-xxx",
    "document_name": "产品手册.pdf"
  }
}
```

**特点**:
- 彻底删除，无法恢复
- 需要管理员权限（TODO）
- 同时删除物理文件和所有关联数据
- 删除 knowledge_base_documents 关联

## 定期清理任务

### 1. 清理已删除文档

**任务名称**: `cleanup_deleted_documents`

**执行时间**: 每天凌晨 2:00

**功能**: 清理已删除超过 30 天的文档

**配置**:
```python
# tasks/celery_tasks.py
"cleanup-deleted-documents": {
    "task": "cleanup_deleted_documents",
    "schedule": crontab(hour=2, minute=0),
    "kwargs": {"days": 30},  # 可调整天数
}
```

**清理流程**:
1. 查询已删除超过 N 天的文档
2. 删除关联数据（knowledge_base_documents）
3. 删除物理文件
4. 删除文档记录

### 2. 清理临时文件

**任务名称**: `cleanup_temp_files`

**执行时间**: 每天凌晨 4:00

**功能**: 清理超过 7 天的临时文件

**配置**:
```python
"cleanup-temp-files": {
    "task": "cleanup_temp_files",
    "schedule": crontab(hour=4, minute=0),
    "kwargs": {"days": 7},  # 可调整天数
}
```

**清理流程**:
1. 查询 file_key 包含 `/temp/` 的文档
2. 检查是否已关联到知识库
3. 删除未关联的临时文件

## 使用示例

### Python 客户端示例

```python
import httpx

# 1. 软删除文档
async def soft_delete_document(document_id: str, token: str):
    async with httpx.AsyncClient() as client:
        response = await client.post(
            "http://localhost:8200/api/v1/documents/soft-delete",
            json={"document_id": document_id},
            headers={"Authorization": f"Bearer {token}"}
        )
        return response.json()

# 2. 查看回收站
async def list_recycle_bin(token: str, page: int = 1):
    async with httpx.AsyncClient() as client:
        response = await client.get(
            f"http://localhost:8200/api/v1/documents/recycle-bin?page={page}&page_size=20",
            headers={"Authorization": f"Bearer {token}"}
        )
        return response.json()

# 3. 恢复文档
async def restore_document(document_id: str, token: str):
    async with httpx.AsyncClient() as client:
        response = await client.post(
            "http://localhost:8200/api/v1/documents/restore",
            json={"document_id": document_id},
            headers={"Authorization": f"Bearer {token}"}
        )
        return response.json()

# 4. 物理删除文档
async def permanent_delete_document(document_id: str, token: str):
    async with httpx.AsyncClient() as client:
        response = await client.post(
            "http://localhost:8200/api/v1/documents/permanent-delete",
            json={"document_id": document_id},
            headers={"Authorization": f"Bearer {token}"}
        )
        return response.json()
```

### 查询未删除的文档

```python
from sqlalchemy import select
from models.document import Document

# 方式1：手动过滤
stmt = select(Document).where(
    Document.tenant_id == tenant_id,
    Document.is_deleted == False
)

# 方式2：使用 CRUD 工厂（自动过滤）
# CRUD 工厂会自动过滤 is_deleted = False 的记录
```

### 查询已删除的文档

```python
stmt = select(Document).where(
    Document.tenant_id == tenant_id,
    Document.is_deleted == True
).order_by(Document.deleted_at.desc())
```

### 查询需要清理的文档

```python
from datetime import datetime, timedelta

cutoff_date = datetime.utcnow() - timedelta(days=30)

stmt = select(Document).where(
    Document.is_deleted == True,
    Document.deleted_at < cutoff_date
)
```

## 性能优化

### 1. 使用部分索引

部分索引只索引未删除的文档，大幅减少索引大小：

```sql
CREATE INDEX idx_docs_active ON documents (tenant_id, owner_id) 
WHERE is_deleted = false;
```

### 2. 查询优化

```python
# ✅ 推荐：使用 is_deleted 字段
stmt = select(Document).where(
    Document.tenant_id == tenant_id,
    Document.is_deleted == False
)

# ❌ 不推荐：使用 deleted_at IS NULL
stmt = select(Document).where(
    Document.tenant_id == tenant_id,
    Document.deleted_at.is_(None)
)
```

### 3. 定期清理

定期清理已删除的文档，避免表数据量过大：
- 已删除超过 30 天的文档自动物理删除
- 临时文件超过 7 天自动清理

## 注意事项

1. **软删除是默认行为**：用户删除文档时，默认使用软删除
2. **物理删除需谨慎**：物理删除后无法恢复，建议只有管理员可以执行
3. **定期清理**：已删除超过 30 天的文档会自动物理删除
4. **权限控制**：只有文档所有者可以删除和恢复文档
5. **关联数据**：物理删除时会同时删除关联的 knowledge_base_documents 记录
