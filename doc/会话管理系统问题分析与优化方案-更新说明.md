## 🎯 核心变更对比

### 原方案（错误）
```typescript
// ❌ 错误：使用了不存在的 isAuthenticated 字段
clearAuth: () => {
  set({ user: null, isAuthenticated: false })
  localStorage.removeItem('auth-storage')
}
```

### 新方案（正确）
```typescript
// ✅ 正确：使用实际的状态结构
clearAuth: () => {
  console.log('[AuthStore] Clearing auth state (no API call)...')
  
  // 清理认证状态（包括 access_token）
  set({
    user: null,
    roles: [],
    permissions: [],
    access_token: null
  })
  
  console.log('[AuthStore] Auth state cleared')
}

// 🔥 同时清理 Permission Store
const { usePermissionStore } = await import('@/stores/permission-store')
usePermissionStore.getState().clearMenus()
```

## 📊 更新后的完整流程

### 场景 1：Token 刷新失败（自动）
```
收到 401 错误
  ↓
尝试刷新 Token
  ↓
刷新失败（session 已过期）
  ↓
调用 clearAuth()（只清除本地状态）
  ↓
同时清除 Permission Store
  ↓
显示提示并跳转登录页
```

### 场景 2：用户主动登出
```
用户点击"退出登录"
  ↓
调用 logout()（调用后端接口）
  ↓
后端清除 Cookie
  ↓
前端清除本地状态
  ↓
清除 Permission Store
  ↓
跳转登录页
```
