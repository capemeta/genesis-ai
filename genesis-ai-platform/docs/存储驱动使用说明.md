# 存储驱动使用说明

## 核心原则

**以 `documents` 表中的字段为准，而不是全局配置**

这样设计的原因：
1. 支持存储迁移（旧文件用旧路径，新文件用新路径）
2. 支持混合存储（部分文件在本地，部分在 S3）
3. 支持多存储位置（不同租户使用不同存储）

## 字段说明

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `storage_driver` | String | 存储驱动类型 | `local` / `s3` |
| `bucket_name` | String | 存储位置<br>- 本地存储：基础路径<br>- S3 存储：bucket 名称 | 本地：`./storage-data`<br>S3：`genesis-ai-files` |
| `file_key` | String | 相对路径（相对于 bucket_name） | `tenant-uuid/documents/2026/02/doc-uuid.txt` |

## 使用场景

### 1. 上传新文件（使用全局配置）

```python
# api/v1/documents.py - upload_document()

# ✅ 正确：使用全局配置（当前系统配置）
bucket_name = (
    settings.SEAWEEDFS_BUCKET if settings.STORAGE_DRIVER == "s3"
    else settings.LOCAL_STORAGE_PATH
)

new_document = Document(
    storage_driver=settings.STORAGE_DRIVER,  # 当前配置
    bucket_name=bucket_name,                 # 当前配置
    file_key=file_key
)
```

**原因**：新上传的文件应该使用当前系统配置的存储位置。

### 2. 下载/读取文件（使用文档记录）

```python
# api/v1/documents.py - download_document()

# ✅ 正确：使用文档记录中的配置
if document.storage_driver == "local":
    from core.storage.local_driver import get_local_driver
    storage_driver = get_local_driver(document.bucket_name)  # 文档记录中的路径
elif document.storage_driver == "s3":
    from core.storage.s3_driver import get_s3_driver
    storage_driver = get_s3_driver()
else:
    # 兜底：使用全局配置
    storage_driver = get_storage_driver()

file_content = await storage_driver.get_content(document.file_key)
```

**原因**：文件可能是在旧配置下上传的，必须使用当时的存储位置才能找到文件。

### 3. 删除文件（使用文档记录）

```python
# api/v1/documents.py - permanent_delete_document()

# ✅ 正确：使用文档记录中的配置
if document.storage_driver == "local":
    from core.storage.local_driver import get_local_driver
    storage_driver = get_local_driver(document.bucket_name)
elif document.storage_driver == "s3":
    from core.storage.s3_driver import get_s3_driver
    storage_driver = get_s3_driver()
else:
    storage_driver = get_storage_driver()

await storage_driver.delete(document.file_key)
```

**原因**：同下载，必须使用文件上传时的存储位置。

### 4. 解析文件（使用文档记录）

```python
# rag/chunking/tasks/parse_task.py

# ✅ 正确：使用文档记录中的配置
if doc.storage_driver == "local":
    from core.storage.local_driver import get_local_driver
    storage_driver = get_local_driver(doc.bucket_name)
    data = await storage_driver.get_content(doc.file_key)
elif doc.storage_driver == "s3":
    from core.storage.s3_driver import get_s3_driver
    storage_driver = get_s3_driver()
    data = await storage_driver.get_content(doc.file_key)
else:
    storage_driver = get_storage_driver()
    data = await storage_driver.get_content(doc.file_key)
```

**原因**：解析任务可能在文件上传很久之后才执行，必须使用文件上传时的存储位置。

## 存储迁移场景

### 场景 1：从本地迁移到 S3

```python
# 1. 修改配置（.env）
STORAGE_DRIVER=s3
SEAWEEDFS_BUCKET=genesis-ai-files

# 2. 新上传的文件会使用 S3
# 旧文件仍然在本地，因为 documents 表中记录了：
# - storage_driver = "local"
# - bucket_name = "./storage-data"

# 3. 下载/解析时会自动使用正确的存储位置
```

### 场景 2：更改本地存储路径

```python
# 1. 修改配置（.env）
LOCAL_STORAGE_PATH=/new/storage/path

# 2. 新上传的文件会使用新路径
# 旧文件仍然在旧路径，因为 documents 表中记录了：
# - storage_driver = "local"
# - bucket_name = "./storage-data"  # 旧路径

# 3. 下载/解析时会自动使用正确的路径
```

### 场景 3：混合存储

```python
# 不同租户使用不同存储
# - 租户 A：本地存储（./storage-data）
# - 租户 B：S3 存储（genesis-ai-files）

# 每个文档的 storage_driver 和 bucket_name 都不同
# 系统会自动使用正确的存储驱动
```

## 错误示例

### ❌ 错误：使用全局配置读取文件

```python
# 错误示例
if settings.STORAGE_DRIVER == "local":
    file_path = os.path.join(settings.LOCAL_STORAGE_PATH, doc.file_key)
    # 问题：如果文件是在旧配置下上传的，这里会找不到文件
```

### ✅ 正确：使用文档记录读取文件

```python
# 正确示例
if doc.storage_driver == "local":
    storage_driver = get_local_driver(doc.bucket_name)
    data = await storage_driver.get_content(doc.file_key)
    # 正确：使用文件上传时的存储位置
```

## 总结

| 操作 | 使用配置 | 原因 |
|------|---------|------|
| 上传新文件 | 全局配置（`settings.STORAGE_DRIVER`） | 使用当前系统配置 |
| 下载文件 | 文档记录（`document.storage_driver`） | 使用文件上传时的配置 |
| 删除文件 | 文档记录（`document.storage_driver`） | 使用文件上传时的配置 |
| 解析文件 | 文档记录（`doc.storage_driver`） | 使用文件上传时的配置 |

**核心原则**：
- **写入（上传）**：使用全局配置
- **读取（下载/解析/删除）**：使用文档记录中的配置

这样设计可以无缝支持存储迁移和混合存储！
