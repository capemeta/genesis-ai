## 二、路由系统详解

### 2.1 TanStack Router 特性

本项目使用 **TanStack Router**，这是一个现代化的、类型安全的路由解决方案，具有以下特点：

1. **文件系统路由** - 基于文件结构自动生成路由
2. **类型安全** - 完整的 TypeScript 类型推断
3. **自动代码分割** - 按路由自动拆分代码
4. **搜索参数验证** - 使用 Zod 验证 URL 参数
5. **嵌套布局** - 支持多层嵌套的布局组件
6. **预加载策略** - 智能预加载路由数据

### 2.2 路由文件命名规则

TanStack Router 使用特殊的文件命名约定：

| 文件名格式 | 说明 | 示例 |
|-----------|------|------|
| `__root.tsx` | 根路由，所有路由的顶层 | `src/routes/__root.tsx` |
| `index.tsx` | 索引路由，对应父路径 | `/tasks/index.tsx` → `/tasks` |
| `route.tsx` | 布局路由，不生成路径 | `/_authenticated/route.tsx` |
| `_layout/` | 下划线前缀 = 布局路由（不影响 URL） | `/_authenticated/` |
| `(group)/` | 括号包裹 = 路由组（不影响 URL） | `/(auth)/sign-in.tsx` → `/sign-in` |
| `$param.tsx` | 美元符号 = 动态参数 | `/users/$id.tsx` → `/users/123` |

### 2.4 路由生成机制

1. **自动生成** - Vite 插件 `@tanstack/router-plugin` 监听文件变化
2. **生成文件** - `src/routeTree.gen.ts` 自动生成路由树
3. **类型安全** - 自动生成 TypeScript 类型定义
4. **热更新** - 开发时自动重新生成

### 2.5 布局嵌套层级

```
__root.tsx (根布局)
└── _top-nav/route.tsx (顶部导航布局)
    └── settings/route.tsx (设置侧边栏布局)
        └── profile.tsx (具体页面)
```

最终渲染结构：
```tsx
<Root>
  <TopNavLayout>
    <SettingsLayout>
      <SettingsProfile />
    </SettingsLayout>
  </TopNavLayout>
</Root>
```

### 案例 2：添加带参数的页面

**需求**：添加文档详情页 `/knowledge-base/:folderId/documents/:docId`

**步骤**：

1. 创建文件 `src/routes/_top-nav/knowledge-base/$folderId.documents.$docId.tsx`

```tsx
import { createFileRoute } from '@tanstack/react-router'
import { useQuery } from '@tanstack/react-query'
import { fetchDocument } from '@/lib/api/documents'

export const Route = createFileRoute('/_top-nav/knowledge-base/$folderId/documents/$docId')({
  component: DocumentDetail,
})

function DocumentDetail() {
  const { folderId, docId } = Route.useParams()
  
  const { data: document, isLoading } = useQuery({
    queryKey: ['document', folderId, docId],
    queryFn: () => fetchDocument(folderId, docId),
  })

  if (isLoading) return <div>加载中...</div>

  return (
    <div className="container p-6">
      <h1 className="text-2xl font-bold">{document?.name}</h1>
      <p className="text-muted-foreground">文件夹: {folderId}</p>
      <div className="mt-4">{document?.content}</div>
    </div>
  )
}
```

2. 访问 `http://localhost:5173/knowledge-base/folder-123/documents/doc-456`

### 案例 3：添加带搜索参数验证的页面

**需求**：添加产品列表页，支持分页和筛选

**步骤**：

1. 创建文件 `src/routes/_top-nav/products/index.tsx`

```tsx
import { z } from 'zod'
import { createFileRoute } from '@tanstack/react-router'
import { useQuery } from '@tanstack/react-query'
import { fetchProducts } from '@/api/products'

// 定义搜索参数 schema
const productSearchSchema = z.object({
  page: z.number().optional().catch(1),
  pageSize: z.number().optional().catch(20),
  category: z.string().optional(),
  sort: z.enum(['price', 'name', 'date']).optional().catch('date'),
})

export const Route = createFileRoute('/_top-nav/products/')({
  validateSearch: productSearchSchema,
  component: ProductList,
})

function ProductList() {
  // 类型安全的搜索参数
  const { page, pageSize, category, sort } = Route.useSearch()
  
  const { data } = useQuery({
    queryKey: ['products', { page, pageSize, category, sort }],
    queryFn: () => fetchProducts({ page, pageSize, category, sort }),
  })

  return (
    <div className="container p-6">
      <h1 className="text-2xl font-bold mb-4">产品列表</h1>
      {/* 产品列表渲染 */}
    </div>
  )
}
```

### 4.1 文件组织

```
src/
├── routes/                    # 路由定义（仅路由配置）
│   └── _top-nav/
│       └── products/
│           └── index.tsx      # 路由配置 + 简单组件
│
├── features/                  # 功能模块（复杂业务逻辑）

│
└── components/                # 通用组件
    └── layout/
```

### 4.2 代码分离原则

- **简单页面**：直接在路由文件中定义组件
- **复杂页面**：在 `features/` 目录创建独立模块
- **布局组件**：在 `components/layout/` 目录定义
- **API 调用**：在 `features/*/api.ts` 或 `lib/api/` 中定义

### 4.3 类型安全

```tsx
// 利用 TanStack Router 的类型推断
function MyComponent() {
  const params = Route.useParams()  // 自动推断参数类型
  const search = Route.useSearch()  // 自动推断搜索参数类型
  const navigate = Route.useNavigate()  // 类型安全的导航
  
  // 类型安全的导航
  navigate({
    to: '/products/$id',
    params: { id: '123' },
    search: { tab: 'details' },
  })
}
```

### 4.4 数据预加载

```tsx
export const Route = createFileRoute('/_top-nav/products/$id')({
```

## 五、常见问题

### Q1: 路由文件修改后没有生效？

**A**: 检查 `src/routeTree.gen.ts` 是否自动更新，如果没有：
- 重启开发服务器 `pnpm dev`
- 检查文件命名是否符合规范
- 查看控制台是否有错误信息

### Q2: 如何实现路由权限控制？

**A**: 使用 `beforeLoad` 钩子：

```tsx
export const Route = createFileRoute('/_top-nav/admin')({
  beforeLoad: async ({ context }) => {
    const { isAuthenticated, hasPermission } = context.auth
    
    if (!isAuthenticated) {
      throw redirect({ to: '/sign-in' })
    }
    
    if (!hasPermission('admin')) {
      throw redirect({ to: '/403' })
    }
  },
})
```

### Q3: 如何在路由间传递状态？

**A**: 使用搜索参数或状态管理：

```tsx
// 方式 1: 搜索参数
navigate({
  to: '/products',
  search: { from: 'dashboard', filter: 'active' },
})

// 方式 2: Zustand 状态管理
const useStore = create((set) => ({
  selectedProduct: null,
  setSelectedProduct: (product) => set({ selectedProduct: product }),
}))
```

### Q4: 如何处理 404 页面？

**A**: 在 `__root.tsx` 中配置：

```tsx
export const Route = createRootRouteWithContext()({
  component: RootLayout,
  notFoundComponent: NotFoundError,  // 404 组件
})
```

## 八、项目特定说明

项目包含两套认证示例：

1. **自定义认证** (`(auth)/login.tsx` 等)
   - 使用后端 Session Token 认证
   - 支持登录、注册、忘记密码、OTP 验证
   - 推荐用于生产环境

2. **Clerk 认证** (`clerk/` 目录)
   - 第三方认证服务示例
   - 仅供参考，可选使用

### 8.3 开发建议

1. **新增业务页面**：优先在 `_top-nav/` 下创建，使用统一的顶部导航布局
2. **示例参考**：`_authenticated/` 目录包含多个示例页面，可作为参考
3. **动态路由**：使用 `$paramName.tsx` 格式创建动态路由
4. **嵌套路由**：使用 `route.tsx` 创建布局，子页面自动继承布局
