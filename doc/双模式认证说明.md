# 双模式认证说明

## 概述

为了支持多端（Web、移动端、API），认证系统同时支持两种认证方式：

| 模式 | 适用场景 | 安全性 | 当前使用 |
|------|---------|--------|---------|
| **Cookie 模式** | Web 浏览器 | ⭐⭐⭐⭐⭐ 最高 | ✅ Web 前端 |
| **Bearer Token 模式** | 移动端、API | ⭐⭐⭐⭐ 高 | 未来支持 |

## Cookie 模式（当前使用）

### 特点

- ✅ **HttpOnly Cookie**：JavaScript 无法读取，防止 XSS 攻击
- ✅ **SameSite=Lax**：防止 CSRF 攻击
- ✅ **浏览器自动携带**：前端无需手动处理
- ✅ **自动过期管理**：浏览器自动处理 Cookie 过期

### 实现

**后端（登录）**：
```python
# 设置 HttpOnly Cookie
response.set_cookie(
    key="access_token",
    value=session_data["access_token"],
    httponly=True,      # 防止 XSS
    samesite="lax",     # 防止 CSRF
    secure=is_https,    # HTTPS 环境启用
    max_age=expires_in,
)
```

**后端（认证中间件）**：
```python
# 优先从 Cookie 读取
session_id = request.cookies.get("access_token")
if not session_id:
    # 兼容 Bearer Token 模式
    auth_header = request.headers.get("Authorization")
    if auth_header:
        session_id = auth_header[7:]  # 去掉 "Bearer "
```

**前端（Axios 配置）**：
```typescript
const axiosInstance = axios.create({
  withCredentials: true,  // 自动携带 Cookie
})
```

**前端（登录）**：
```typescript
// 调用登录 API，后端自动设置 Cookie
await login({ username, password, remember_me: true })

// 不需要存储 token
// 浏览器会自动管理 Cookie
```

## Bearer Token 模式（未来支持）

### 特点

- ✅ **灵活性高**：适合移动端和第三方 API
- ✅ **跨平台**：支持所有客户端
- ✅ **无 CORS 限制**：简化跨域配置
- ⚠️ **需防范 XSS**：token 存储在客户端

### 实现（未来）

**后端（已支持）**：
```python
# 登录时返回 token
return Token(**session_data)

# 认证中间件已支持从 Header 读取
auth_header = request.headers.get("Authorization")
if auth_header and auth_header.startswith("Bearer "):
    session_id = auth_header[7:]
```

**移动端（未来）**：
```typescript
// 登录
const response = await login({ username, password })
await SecureStorage.setItem('access_token', response.access_token)

// 请求
const token = await SecureStorage.getItem('access_token')
headers: {
  'Authorization': `Bearer ${token}`
}
```

## 安全性对比

| 特性 | Cookie 模式 | Bearer Token 模式 |
|------|------------|------------------|
| **XSS 防护** | ✅ HttpOnly 完全防护 | ⚠️ 需要防范 XSS |
| **CSRF 防护** | ✅ SameSite 防护 | ✅ 不受 CSRF 影响 |
| **中间人攻击** | ✅ HTTPS + Secure | ✅ HTTPS 保护 |
| **实现复杂度** | ✅ 简单（浏览器自动） | ⚠️ 中等（需手动管理） |
| **跨域支持** | ⚠️ 需要 CORS 配置 | ✅ 简单 |
| **适用场景** | Web 浏览器 | 移动端、API |

## 为什么 Web 使用 Cookie 模式？

1. **最安全**：HttpOnly Cookie 完全防止 XSS 窃取 token
2. **最简单**：浏览器自动处理，前端无需存储和管理 token
3. **行业标准**：Google、Facebook、GitHub 等都使用 Cookie 认证
4. **用户体验好**：无感知的认证和刷新

## 为什么移动端使用 Bearer Token 模式？

1. **灵活性**：移动端没有 Cookie 机制
2. **跨平台**：iOS、Android、小程序都支持
3. **简单**：不需要处理 CORS
4. **可控性**：应用完全控制 token 存储

## 切换模式

### Web 前端切换到 Bearer Token 模式

如果需要切换（不推荐）：

1. 登录时存储 token 到 localStorage
2. 请求时添加 Authorization Header
3. 登出时清除 localStorage

### 移动端使用 Bearer Token 模式

1. 使用 SecureStorage 存储 token
2. 请求时添加 Authorization Header
3. 登出时清除 SecureStorage
