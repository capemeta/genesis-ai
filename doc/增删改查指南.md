## 后端开发

### 1. 模型定义

#### 1.1 基础模型结构

```python
# models/knowledge_base.py
from sqlalchemy import String, Text
from sqlalchemy.orm import Mapped, mapped_column
from models.base import Base, AuditMixin
from uuid import UUID

class KnowledgeBase(Base, AuditMixin):
    """
    知识库模型
    
    继承 AuditMixin 自动获得审计字段：
    - owner_id, created_by_id, updated_by_id
    - created_by_name, updated_by_name
    - created_at, updated_at
    """
    __tablename__ = "knowledge_bases"
    
    # 🔍 配置可搜索字段（用于 search 参数）
    __searchable_fields__ = ["name", "description"]
    
    # 租户隔离（必需）
    tenant_id: Mapped[UUID] = mapped_column(nullable=False, index=True)
    
    # 业务字段
    name: Mapped[str] = mapped_column(String(255), nullable=False)
    description: Mapped[str | None] = mapped_column(Text, nullable=True)
    visibility: Mapped[str] = mapped_column(String(20), default="private")
```

#### 1.2 配置搜索字段

```python
# 不同模型的搜索字段配置示例

# 用户：搜索用户名、邮箱、昵称

# 文档：搜索标题和内容
```

```python
#### 2.2 自动生成的标准路由
后端会自动生成以下符合项目规范的路由（只使用 GET 和 POST 两种请求方式，禁止使用路径参数）：

| 功能 | HTTP 方法 | 路径 | 参数位置 | 参数示例 |
| :--- | :--- | :--- | :--- | :--- |
| **列表** | `GET` | `/list` | Query | `?page=1&page_size=20&search=...` |
| **详情** | `GET` | `/get` | Query | `?id=uuid-...` |
| **创建** | `POST` | `/create` | Body | `{"name": "...", ...}` |
| **更新** | `POST` | `/update` | Body | `{"id": "uuid-...", "name": "..."}` |
| **删除** | `POST` | `/delete` | Body | `{"id": "uuid-..."}` |

> **规范要点**：
> - GET 用于查询操作（列表、详情、搜索等）
> - POST 用于增删改操作（创建、更新、删除等）
> - 禁止使用路径参数（如 `/{id}`），ID 通过查询参数或请求体传递
> - URL 路径保持干净，安全性更高，前端封装更一致
```

### 3. Schema 定义（可选）

如果需要自定义验证规则，可以定义 Schema：

```python
# schemas/knowledge_base.py
from pydantic import BaseModel, Field, field_validator
from uuid import UUID
from datetime import datetime

class KnowledgeBaseCreate(BaseModel):
    """创建知识库"""
    name: str = Field(..., min_length=1, max_length=255)
    description: str | None = None
    visibility: str = Field(default="private")
    
    @field_validator("name")
    @classmethod
    def validate_name(cls, v: str) -> str:
        if not v.strip():
            raise ValueError("名称不能为空")
        return v.strip()

class KnowledgeBaseUpdate(BaseModel):
    """更新知识库"""
    name: str | None = Field(None, min_length=1, max_length=255)
    description: str | None = None
    visibility: str | None = None

class KnowledgeBaseRead(BaseModel):
    """知识库响应"""
    id: UUID
    tenant_id: UUID
    owner_id: UUID
    name: str
    description: str | None
    visibility: str
    created_by_name: str
    created_at: datetime
    updated_at: datetime
    
    class Config:
        from_attributes = True

# 注册时使用自定义 Schema
crud_factory.register(
    model=KnowledgeBase,
    prefix="/knowledge-bases",
    tags=["knowledge-bases"],
```

### 4. 自定义 Service（可选）

如果需要复杂业务逻辑，可以自定义 Service：

```python
# services/knowledge_base_service.py
from core.base_service import BaseService
from models.knowledge_base import KnowledgeBase
from schemas.knowledge_base import KnowledgeBaseCreate, KnowledgeBaseUpdate

class KnowledgeBaseService(BaseService[KnowledgeBase, KnowledgeBaseCreate, KnowledgeBaseUpdate]):
    """知识库服务"""
    
    async def create(self, data: KnowledgeBaseCreate, current_user):
        """重写创建方法，添加自定义逻辑"""
        # 自定义验证
        if await self._name_exists(data.name, current_user.tenant_id):
            raise HTTPException(400, "知识库名称已存在")
        
        # 调用父类方法
        kb = await super().create(data, current_user)
        
        # 创建后的额外操作
        await self._create_default_folder(kb.id)
        
        return kb
    
    async def _name_exists(self, name: str, tenant_id: UUID) -> bool:
        """检查名称是否存在"""
        stmt = select(KnowledgeBase).where(
            KnowledgeBase.tenant_id == tenant_id,
            KnowledgeBase.name == name,
            KnowledgeBase.deleted_at.is_(None)
        )
        result = await self.db.execute(stmt)
        return result.scalar_one_or_none() is not None

# 注册时使用自定义 Service
crud_factory.register(
    model=KnowledgeBase,
    service_class=KnowledgeBaseService,
    prefix="/knowledge-bases",
    tags=["knowledge-bases"]
)
```

## 过滤与搜索

### 三种过滤模式

#### 1. 简单搜索（search）

**用途**：用户输入的搜索框，模糊匹配

**特点**：
- 模糊匹配（不区分大小写）
- 搜索模型定义的 `__searchable_fields__` 字段
- 多个字段之间是 OR 关系

**后端配置**：
```python
class Tenant(Base):
    __searchable_fields__ = ["name", "description"]  # 搜索这两个字段
```

**前端使用**：
```typescript
const result = await fetchTenants({
  search: "测试",  // 模糊搜索 name 或 description
  page: 1,
  page_size: 20
})
```

**生成的 SQL**：
```sql
WHERE (name ILIKE '%测试%' OR description ILIKE '%测试%')
```


#### 2. 精确过滤（filters）

**用途**：下拉框、标签等精确匹配场景

**特点**：
- 精确匹配（`==`）
- 字典格式，键值对
- 多个条件之间是 AND 关系

**前端使用**：
```typescript
const result = await fetchTenants({
  filters: {
    owner_id: "uuid-xxx",
    status: "active"  // 精确匹配
  },
  page: 1,
  page_size: 20
})
```

**生成的 SQL**：
```sql
WHERE owner_id = 'uuid-xxx' AND status = 'active'
```

#### 3. 高级过滤（advanced_filters）

**用途**：复杂查询，支持多种操作符

**支持的操作符**：

| 操作符 | 说明 | 示例 |
|--------|------|------|
| `eq` | 等于 | `{"field": "name", "op": "eq", "value": "测试"}` |
| `ne` | 不等于 | `{"field": "status", "op": "ne", "value": "deleted"}` |
| `gt` | 大于 | `{"field": "max_users", "op": "gt", "value": 10}` |
| `gte` | 大于等于 | `{"field": "created_at", "op": "gte", "value": "2024-01-01"}` |
| `lt` | 小于 | `{"field": "max_users", "op": "lt", "value": 100}` |
| `lte` | 小于等于 | `{"field": "created_at", "op": "lte", "value": "2024-12-31"}` |
| `like` | 模糊匹配 | `{"field": "name", "op": "like", "value": "测试"}` |
| `in` | 在列表中 | `{"field": "status", "op": "in", "value": ["active", "pending"]}` |
| `not_in` | 不在列表中 | `{"field": "status", "op": "not_in", "value": ["deleted"]}` |
| `is_null` | 为空 | `{"field": "deleted_at", "op": "is_null"}` |
| `is_not_null` | 不为空 | `{"field": "description", "op": "is_not_null"}` |

**前端使用**：
```typescript
const result = await fetchTenants({
  advanced_filters: [
    { field: "name", op: "like", value: "测试" },
    { field: "max_users", op: "gt", value: 10 },
    { field: "created_at", op: "gte", value: "2024-01-01" }
  ],
  page: 1,
  page_size: 20
})
```

**生成的 SQL**：
```sql
WHERE name ILIKE '%测试%' 
  AND max_users > 10 
  AND created_at >= '2024-01-01'
```

### 组合使用

三种模式可以同时使用，它们之间是 AND 关系：

```typescript
// 前端示例：组合使用
const result = await fetchTenants({
  search: "测试",  // 模糊搜索 name 或 description
  filters: {
    owner_id: "uuid-xxx"  // 精确匹配 owner_id
  },
  advanced_filters: [
    { field: "max_users", op: "gt", value: 10 }  // max_users > 10
  ],
  page: 1,
  page_size: 20
})
```

**生成的 SQL**：
```sql
WHERE (name ILIKE '%测试%' OR description ILIKE '%测试%')  -- search
  AND owner_id = 'uuid-xxx'                                -- filters
  AND max_users > 10                                       -- advanced_filters
```

### 5. 自定义与覆盖默认功能

当自动生成的 CRUD 路由无法满足特定业务需求时，你可以通过以下两种方式进行自定义：

#### 5.1 方式一：覆盖业务逻辑（推荐）
如果你只是想改变接口内部的处理逻辑（例如：租户删除需要硬删除，或者创建资源后需要发送通知），只需**编写自定义 Service 并重写对应方法**。

```python
# 1. 在自定义 Service 中重写基类方法
class TenantService(BaseService):
    async def delete(self, resource_id, current_user, **kwargs):
        # 执行自定义逻辑（如强制硬删除）
        return await super().delete(resource_id, current_user, soft_delete=False)

# 2. 在注册时指定该 Service
crud_factory.register(
    model=Tenant,
    service_class=TenantService,  # 注入你的自定义 Service
    ...
)
```

#### 5.2 方式二：完全覆盖接口路由

如果你想彻底改变接口的参数结构、路径或行为逻辑。

1.  在注册时**禁用**该默认路由：
    ```python
    crud_factory.register(
        model=Tenant,
        enable_delete=False,  # 禁用默认生成的 POST /delete 接口
        ...
    )
    ```
2.  获取模块 `router` 并**手动定义**同名接口：
    ```python
    # 获取已经配置好 prefix 和 tags 的 router 对象
    router = crud_factory.get_router("Tenant")
    
    @router.post("/delete")
    async def custom_delete(request: MyCustomSchema):
        # 编写完全自定义的业务代码
        ...
    ```

> **总结**：只是改内部逻辑推荐用 **Service 重写**；如果要改接口参数或完全重做推荐用 **禁用+手写**。

#### 通用类型定义

```typescript
// src/lib/api/types.ts - 所有 API 共用
export interface ListRequest {
  page?: number
  page_size?: number
  search?: string
  filters?: Record<string, any>
  advanced_filters?: AdvancedFilter[]
  sort_by?: string
  sort_order?: 'asc' | 'desc'
}

export interface PaginatedResponse<T> {
  data: T[]
  total: number
}
```

#### 通用 Hook

```typescript
// src/hooks/use-list-query.ts
export function useListQuery<T>(
  queryKey: string[],
  fetchFn: (params: ListRequest) => Promise<PaginatedResponse<T>>,
  params: ListRequest = {}
) {
  return useQuery({
    queryKey: [...queryKey, params],
    queryFn: () => fetchFn(params)
  })
}

// 使用
const { data } = useListQuery(
  ['tenants'],
  fetchTenants,
  { search, filters: { owner_id } }
)
```
