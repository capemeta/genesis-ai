## 如何检查连线是否正常

### 检查步骤：

1. **查看左上角的统计面板**
   - 显示"连接: 3"表示初始有3条边
   - 每次连接后数字应该增加

2. **查看浏览器控制台**
   - 连接节点时应该看到：
     ```
     🔗 创建新连接: { source: "1", target: "2", type: "smoothstep", ... }
     📊 当前所有边: [...]
     ```

3. **测试连接**
   - 从节点**右侧**的蓝色圆点开始拖拽
   - 拖到另一个节点**左侧**的蓝色圆点
   - 释放鼠标，应该看到：
     - Toast 提示"节点已连接"
     - 控制台输出调试信息
     - 左上角连接数增加

4. **查看初始连接线**
   - 页面加载后应该有3条连接线：
     - 节点 1 → 节点 2
     - 节点 2 → 节点 3
     - 节点 3 → 节点 4（带动画）

## 可能的问题和解决方案

### 问题1: 完全看不到连接线
**检查:**
- 打开浏览器开发者工具 (F12)
- 查看 Elements 标签
- 查找 `<svg class="react-flow__edges">`
- 检查内部是否有 `<path>` 元素

**如果没有 path 元素:**
- 可能是 ReactFlow 版本不兼容
- 检查 package.json 中的 reactflow 版本

### 问题2: 连接线很淡或半透明
**解决方法:**
- 已在 CSS 中增强颜色透明度
- 使用明显的颜色 (蓝色)
- 如果还是看不清，可以在 `defaultEdgeOptions` 中使用更亮的颜色

### 问题3: 连接手柄看不到
**检查:**
- 节点左右两侧应该有蓝色的圆点
- 有光晕效果
- 如果看不到，可能是 CSS 没有加载

### 问题4: 可以创建连接但看不到线
**调试:**
1. 打开控制台看是否有报错
2. 检查 edge 数据结构是否正确
3. 尝试缩小画布，看看是否在视野外

## 样式配置

### 边的默认样式
```javascript
defaultEdgeOptions={{
  type: 'smoothstep',
  animated: false,
  style: { 
    strokeWidth: 2, 
    stroke: '#64748b' // slate-500
  },
}}
```

### 新连接的样式
```javascript
style: { 
  strokeWidth: 3, 
  stroke: '#3b82f6' // blue-500
}
```

### CSS 配置
```css
.react-flow__edge-path {
  stroke: hsl(var(--muted-foreground) / 0.8) !important;
  stroke-width: 2 !important;
  fill: none !important;
}
```

## 下一步

如果以上修复还是看不到连接线，请：

1. **刷新页面** (Ctrl+F5 强制刷新)
2. **检查控制台** 是否有错误
3. **查看 Network 标签** 确认 CSS 文件已加载
4. **尝试缩放画布** 看看线是否在视野外
5. **检查主题** 深色/浅色模式切换看是否有区别
